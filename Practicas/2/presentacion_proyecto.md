# Presentación del proyecto

1.º DAW · Proyecto Intermodular · NexHub Coworking · 2025

---

## Qué es NexHub

Un sistema de gestión para un coworking tecnológico en Madrid. Tiene tres partes: una web pública para los clientes, una aplicación Java para el administrador y una base de datos MySQL que conecta ambas.

No es un ejercicio genérico. Está modelado como si fuera un encargo real: la empresa tiene nombre, dirección, precios concretos y un flujo de trabajo que tiene sentido. El administrador puede dar de alta socios, crear reservas, registrar pagos y consultar estadísticas.

---

## El problema que resuelve

Sin una herramienta así, el administrador de un coworking lleva los socios y las reservas en papel o en una hoja de cálculo. Eso funciona cuando tienes diez socios. Con ciento veinte no.

NexHub automatiza lo básico: saber qué espacios están disponibles en un momento dado, registrar una reserva con su precio calculado automáticamente y mantener un historial de pagos.

---

## Para quién es cada parte

La **web** es para quien todavía no es cliente. Puede ver los espacios, los precios y mandar un mensaje de contacto.

La **aplicación Java** es para el administrador. Desde consola puede gestionar socios, reservas, pagos y empleados. Es interna, no está conectada a la web.

---

## Tecnologías y por qué

Sin frameworks en el frontend porque el módulo evaluaba el dominio de HTML, CSS y JavaScript base. Con Java porque es lo que se trabaja en el módulo de Programación. Con MySQL porque es la base de datos del curso. El stack no es el más moderno posible, pero es el que toca aprender este año.

---

## Qué demuestra

Que sé diseñar un sistema desde cero: modelar los datos en E-R, traducirlos a SQL, conectar Java a MySQL con JDBC, organizar el código en capas y construir una web que se ve bien en cualquier dispositivo. Y que sé documentarlo de forma que otra persona pueda entenderlo y ejecutarlo.
